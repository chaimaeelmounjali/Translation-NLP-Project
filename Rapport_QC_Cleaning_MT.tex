\section{Script 1: check\_and\_correct\_mt.py}
\subsection{Role}
Ce script realise un controle qualite multilingue (Darija arabe, Darija arabizi, anglais, MSA), corrige les champs texte via LLM, puis produit:
\begin{itemize}[leftmargin=*]
  \item un CSV corrige,
  \item un rapport JSON detaille,
  \item un checkpoint JSONL pour reprise/resume.
\end{itemize}

\subsection{Architecture technique}
\begin{itemize}[leftmargin=*]
  \item \textbf{Prompting et few-shot}: variable \texttt{FEW\_SHOT\_EXAMPLES} et construction via \texttt{build\_messages}. 
  \item \textbf{Validation schema}: reponse forcee en \texttt{json\_object} puis validee par Pydantic (\texttt{QCResponse}, \texttt{FieldResult}).
  \item \textbf{Rate limiting}: classe \texttt{TokenBucket} pour RPM/TPM.
  \item \textbf{Concurrence}: \texttt{ThreadPoolExecutor} pour traiter plusieurs lignes en parallele.
  \item \textbf{Robustesse}: retries differencies (\texttt{RateLimitError}, \texttt{APIError}, erreurs JSON).
  \item \textbf{Reprise}: checkpoint incremental et saut des lignes deja traitees.
\end{itemize}

\subsection{Parametres CLI majeurs}
\begin{itemize}[leftmargin=*]
  \item chemins: \texttt{--input}, \texttt{--output}, \texttt{--report}, \texttt{--artifacts-dir}, \texttt{--checkpoint}
  \item modele/API: \texttt{--api-key}, \texttt{--env-file}, \texttt{--model}
  \item controle du batch: \texttt{--start-row}, \texttt{--end-row}, \texttt{--max-rows}, \texttt{--only-status}
  \item performance: \texttt{--workers}, \texttt{--rpm}, \texttt{--tpm}, \texttt{--retries}
  \item mode: \texttt{--dry-run}, \texttt{--output-style}
\end{itemize}

\section{Script 2: clean\_silver\_shard\_3\_corrected.py}
\subsection{Role}
Ce script applique un pipeline de nettoyage NLP modulaire sur les 4 colonnes cibles:
\begin{itemize}[leftmargin=*]
  \item \texttt{darija\_arabic}
  \item \texttt{darija\_arabizi}
  \item \texttt{english}
  \item \texttt{modern\_standard\_arabic}
\end{itemize}

\subsection{Nettoyages appliques (detail fonctionnel)}
\subsubsection{Preprocessing global}
\begin{itemize}[leftmargin=*]
  \item suppression artefacts MT globaux: \texttt{<unk>}, \texttt{@-@}
  \item nettoyage HTML/Markdown/URL/mentions
  \item gestion des emojis (suppression par defaut)
  \item normalisation des espaces
\end{itemize}

\subsubsection{Nettoyage par langue}
\textbf{Anglais}
\begin{itemize}[leftmargin=*]
  \item lowercasing
  \item expansion des contractions (ex: \texttt{don't} $\rightarrow$ \texttt{do not})
  \item retrait ponctuation/symboles et stopwords
  \item options: stem/lemma (desactivees par defaut)
\end{itemize}

\textbf{MSA}
\begin{itemize}[leftmargin=*]
  \item normalisation script arabe (diacritiques, tatweel, variantes Alef)
  \item retrait stopwords MSA
  \item options clitiques/stemming leger (desactivees par defaut)
\end{itemize}

\textbf{Darija arabe}
\begin{itemize}[leftmargin=*]
  \item normalisation arabe adaptee Darija
  \item normalisation de variantes lexicales (mapping)
  \item retrait stopwords Darija
\end{itemize}

\textbf{Darija arabizi}
\begin{itemize}[leftmargin=*]
  \item lowercasing
  \item reduction des repetitions de caracteres
  \item nettoyage ponctuation/symboles
  \item suppression des chiffres isoles
  \item options transliteration vers arabe / drop voyelles (desactivees par defaut)
\end{itemize}

\subsubsection{Filtres de lignes}
\begin{itemize}[leftmargin=*]
  \item suppression des lignes ``metadata contamination''
  \item suppression optionnelle des lignes MSA avec marqueurs Darija
  \item suppression optionnelle des lignes de statut \texttt{ERROR}
  \item suppression des lignes avec champ cible vide (apres nettoyage)
\end{itemize}

\subsubsection{Post-clean quality fixes}
\begin{itemize}[leftmargin=*]
  \item suppression de placeholders residuels selon colonne
  \item collapse des tokens adjacents dupliques
  \item recalcul de \texttt{english\_word\_count}
\end{itemize}

\section{Problemes techniques constates}
\begin{enumerate}[leftmargin=*]
  \item \textbf{Timeouts API (QC)}: 4 lignes en echec malgre retries.
  \item \textbf{Perte potentielle de donnees utiles}: en mode strict, 163 lignes retirees car MSA detectee comme Darija-like.
  \item \textbf{Sortie de recheck incoherente}: \path{silver_shard_3.corrected.auto.errors.rechecked.csv} est vide (0 byte) alors que son checkpoint contient 1 entree.
  \item \textbf{Dependance non exploitee explicitement}: \texttt{python-dotenv} est presente dans requirements mais non importee dans les scripts analyses.
\end{enumerate}

\section{Recommandations}
\begin{enumerate}[leftmargin=*]
  \item Pour conserver le code-switching MSA/Darija: lancer le cleaning avec \texttt{--keep-msa-darija-leakage}.
  \item Pour minimiser les pertes: ajouter une option \texttt{--keep-empty-target-rows} (actuellement absente).
  \item Pour stabiliser les runs QC: reduire \texttt{--workers}, augmenter \texttt{--retries}, relancer uniquement \texttt{--only-status ERROR}.
  \item Pour auditabilite: produire un CSV des IDs supprimes par categorie de filtre.
  \item Pour hygiene dependances: supprimer \texttt{python-dotenv} de requirements ou l utiliser explicitement.
\end{enumerate}
